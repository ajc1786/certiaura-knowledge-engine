# DECISION RECORD — CERT-BUILD-0035F

**Decision:** Adopt controlled Product Passport™ publication, lifecycle monitoring and separate marketplace eligibility records.  
**Date:** 2026-07-18  
**Status:** Approved build; canonical effect follows successful repository installation.

## Existing position retained

Builds 0035D and 0035E remain unchanged. Existing Product Passport, Evidence, Marketplace, relationship, asset-register and change-control architecture remains in force.

## Gap addressed

The installed submission and verification controls did not by themselves define how an approved passport is published, monitored, suspended, expired, superseded or separately admitted to the marketplace.

## Decision

Add a lifecycle record, semantic validator, matrices, templates and registers that:

- publish verified claims only within approved scope;
- separate public, member and internal data;
- apply effective and review dates;
- record trigger-led lifecycle events;
- block active publication or listing where critical controls fail;
- retain immutable history; and
- keep marketplace eligibility independent from evidence verification.

## Effect on locked decisions

No locked decision is superseded. This build extends the existing Product Passport System, Evidence Knowledge System, Marketplace System and Project Genesis validation capability.
