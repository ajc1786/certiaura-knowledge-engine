# Decision Record — CERT-BUILD-0036

**Decision:** Close the 0035 lettered sequence at 0035K and start this operationally distinct customer and post-market capability as Build 0036.  
**Status:** Approved by founder instruction (“green”).  
**Date:** 2026-07-19

## Existing decision retained

Installed Builds 0035D–0035K retain their identifiers, titles, repository history and dependency references.

## Reason

The 0035 chain began as a tightly connected Product Passport™ and supplier evidence lifecycle. Customer eligibility, privacy and post-market surveillance are related but operationally distinct integrated capabilities. A new whole build number improves traceability and prevents indefinite suffix growth.

## Numbering rule applied

- New integrated work package: new whole build number.
- Planned subdivisions of one parent package: suffix letters.
- Corrections: version or revision within the installed build unless a new authorised capability is created.
- Dependencies: recorded in metadata rather than implied by suffix.

## No parallel governance

This record amends the application of the existing naming/versioning controls only. It does not create a replacement naming standard or governance architecture.
