# PEPTIDE FLAGSHIP BOOTSTRAP WORKFLOW

## Phase 1 — Asset creation

1. Confirm the peptide / compound name.
2. Confirm scope and priority.
3. Allocate a permanent PKS UAI through the established asset-control process.
4. Create the peptide asset folder if needed.
5. Copy the reusable flagship templates.
6. Replace approved template tokens.

## Phase 2 — Foundation relationships

Identify or create links to:

- Biology Knowledge System assets
- Receptor / target assets
- Condition Knowledge System assets
- Goal Knowledge System assets
- Monitoring Knowledge System assets
- Safety Knowledge System assets

Do not create duplicate objects when a canonical object already exists.

## Phase 3 — Evidence production

Use the existing Evidence Knowledge System standards.

For each study:

1. Confirm source identity.
2. Record source status.
3. Extract study design.
4. Extract population.
5. Extract intervention and comparator.
6. Extract outcomes.
7. Extract safety.
8. Record limitations and sponsor context.
9. Create relationships.
10. Add source-closure state.

## Phase 4 — Consolidation

Assemble:

- Flagship Knowledge Asset
- Evidence Index
- Module Assembly Map
- Claim Control Matrix
- Safety Evidence Matrix
- Monitoring Evidence Matrix
- Evidence-to-Module Map
- Knowledge Graph
- Certification Gap Register
- User Layer Map
- Executive Summary

## Phase 5 — Quality gate

The asset must pass:

- Identifier integrity
- Source integrity
- Relationship integrity
- Claim-control review
- Safety / monitoring linkage
- Certification-status check
- No arbitrary completion percentage
- No hidden unresolved evidence gaps

## Phase 6 — Release

1. Import through Project Genesis.
2. Review all changes.
3. Commit and push.
4. Confirm GitHub Actions passes.
5. Continue scientific production.
