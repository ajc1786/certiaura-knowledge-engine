# PRIMARY SOURCE CLOSURE STANDARD

## Purpose

Prevent a source from being treated as verified merely because:

- a title exists;
- a secondary article quotes it;
- a sponsor result is widely reported;
- a URL is known;
- a publication is believed to exist.

## Closure states

### CLOSED — PRIMARY CAPTURED

Requirements:

1. Primary source identity confirmed.
2. Primary source directly accessed.
3. Relevant study data extracted.
4. Source identifier recorded.
5. Limitations recorded.
6. Evidence Object updated.
7. No unresolved contradiction affecting the extracted claim.

### IDENTIFIED — EXTRACTION BLOCKED

Use when:

- the primary source identity is known;
- the direct source cannot currently be retrieved or fully accessed.

Required treatment:

- Preserve identifier and access attempt.
- Do not call the evidence fully extracted.
- Do not promote secondary-reported numbers to primary-verified status.

### OPEN — PRIMARY NOT CAPTURED

Use when:

- the evidence lead is known;
- the primary document, registry result or sponsor source has not been directly captured.

Required treatment:

- Keep quantitative claims provisional or omit them.
- Maintain a source-retrieval task.

### SUPERSEDED

Use when:

- a stronger primary source has replaced an earlier evidence lead.

Required treatment:

- Preserve historical provenance.
- Point the earlier object to the stronger source.
- Do not silently delete source history.

## Integrity principle

**Source identity is not the same as source extraction.**

**Source extraction is not the same as scientific certification.**
