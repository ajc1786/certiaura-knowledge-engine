# Retatrutide Controlled Conversation Workflow Baseline

**Universal Asset Identifier:** CERT-SYS-000851
**Build:** CERT-BUILD-0044
**Version:** 1.0.0
**Status:** BASELINE
**Knowledge System:** SYS

## Purpose

This asset defines a deterministic multi-turn orchestration layer over the Build 0043 grounded query engine.

## State controls

- `ACTIVE_GROUNDED` for a grounded repository response;
- `ACTIVE_WITH_ABSTENTION` where sufficient canonical evidence is not resolved;
- `SAFETY_BOUNDARY_ENFORCED` for personalised dosing, diagnosis, procurement or treatment-selection requests;
- `IDENTIFIABLE_INPUT_REJECTED` where direct identifiers are detected;
- `LOCKED_URGENT_ROUTING` after an urgent symptom or emergency-routing response;
- `TURN_LIMIT_REACHED` when the bounded session limit is reached.

Once urgent routing is triggered, later turns in that session cannot dilute or replace the urgent instruction. A new session is required after professional assessment.

## Privacy controls

- pseudonymous session and patient references only;
- no request-body logging;
- no database or browser storage;
- maximum request size and bounded turn count;
- loopback-only server binding by default and by policy.

## Provenance

Every eligible answer is generated by the Build 0043 query engine and retains its query hash, retrieval hash, warnings and repository source metadata.
