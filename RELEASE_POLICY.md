# Release Policy

## Commit
A commit is any meaningful repository update.

## Release
A release is a stable project milestone.

Examples:

- v0.1.0 Foundation Repository
- v0.2.0 Metabolic Biology Pack
- v0.3.0 Retatrutide Platinum Asset
- v0.4.0 Evidence Knowledge System

## Release criteria

A release should include:
- Updated Master Asset Register
- Updated Change Log
- Updated Production Dashboard
- All new assets committed
- Repository validation passing
