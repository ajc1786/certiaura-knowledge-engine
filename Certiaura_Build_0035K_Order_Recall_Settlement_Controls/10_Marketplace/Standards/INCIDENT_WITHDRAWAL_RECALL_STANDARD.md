# Incident, Withdrawal and Recall Standard

- Severity drives time-to-triage, command level, restrictions and notification assessment.
- Critical cases immediately block order release for affected scope and create a payment hold.
- Traceability includes upstream source, internal movements, listings, orders, recipients, returns and residual stock.
- Withdrawal is not a substitute for recall where customer-level action is required.
- Recall effectiveness must be evidenced by contact, response, return/destruction and residual-exposure metrics.
- Closure requires human approval, reconciliation, corrective/preventive action linkage and documented residual risk.
